# 基础设施域业务流程

## 1. 用户认证流程
```
登录请求 → 参数验证 → 用户查询 → 密码验证 → 登录限制检查 → JWT生成 → 审计记录 → 返回token
```

### 详细步骤：
1. **参数验证**：检查用户名和密码格式
2. **用户查询**：从数据库查询用户信息
3. **密码验证**：使用bcrypt比较密码哈希
4. **登录限制检查**：
   - 检查账户是否被锁定
   - 检查账户是否被禁用
   - 记录失败尝试，达到阈值自动锁定
5. **JWT生成**：
   - 生成访问token（短期有效）
   - 生成刷新token（长期有效）
6. **审计记录**：记录登录成功事件
7. **返回token**：返回双token和用户信息

## 2. Token刷新流程
```
刷新请求 → 验证refreshToken → 检查黑名单 → 验证用户状态 → 生成新token → 旧token加入黑名单
```

### 详细步骤：
1. **验证refreshToken**：JWT签名和有效期验证
2. **检查黑名单**：确保token未被撤销
3. **验证用户状态**：检查用户是否存在且启用
4. **生成新token**：
   - 新的访问token
   - 新的刷新token
5. **旧token加入黑名单**：防止重放攻击

## 3. 密码修改流程
```
修改请求 → 验证当前密码 → 密码强度检查 → 加密新密码 → 更新数据库 → 清除缓存 → 审计记录
```

### 详细步骤：
1. **验证当前密码**：bcrypt比较
2. **密码强度检查**：
   - 最小长度8字符
   - 包含大小写字母
   - 包含数字
   - 包含特殊字符
3. **加密新密码**：bcrypt加密，强度12
4. **更新数据库**：更新密码和修改时间
5. **清除缓存**：清除用户信息缓存
6. **审计记录**：记录密码修改事件

## 4. 备份恢复流程
```
备份请求 → 创建备份文件 → 压缩（可选） → 加密（可选） → 验证完整性 → 清理旧备份 → 记录历史
```

### 详细步骤：
1. **创建备份文件**：复制数据库文件
2. **压缩**：Gzip压缩减少文件大小
3. **加密**：AES-256-GCM加密备份文件
4. **验证完整性**：
   - 解密备份文件
   - 解压备份文件
   - 检查SQLite完整性
   - 验证关键表存在
5. **清理旧备份**：删除超过保留数量的旧备份
6. **记录历史**：保存备份记录到数据库和文件系统

## 5. 备份恢复流程
```
恢复请求 → 查找备份 → 解密（可选） → 解压（可选） → 完整性验证 → 备份当前数据库 → 恢复备份 → 系统重启
```

### 详细步骤：
1. **查找备份**：从历史记录中查找备份
2. **解密**：如果是加密备份，解密文件
3. **解压**：如果是压缩备份，解压文件
4. **完整性验证**：检查SQLite数据库完整性
5. **备份当前数据库**：恢复前备份当前数据库
6. **恢复备份**：复制备份文件到数据库路径
7. **系统重启**：触发优雅重启，重新加载数据库

## 6. 通知发送流程
```
通知请求 → 保存数据库 → 检查通知渠道 → 并行发送 → 更新状态 → 错误处理
```

### 详细步骤：
1. **保存数据库**：创建通知记录
2. **检查通知渠道**：根据配置启用相应渠道
3. **并行发送**：
   - 企业微信：Markdown消息
   - 钉钉：Markdown消息
   - 邮件：HTML格式
   - Webhook：Socket.IO推送
4. **更新状态**：根据发送结果更新状态
5. **错误处理**：记录错误信息，支持重试

## 7. 配置管理流程
```
配置请求 → 加密存储 → 更新数据库 → 更新预设Agent → 审计记录
```

### 详细步骤：
1. **加密存储**：敏感配置使用凭证服务加密
2. **更新数据库**：使用upsert语句更新配置
3. **更新预设Agent**：自动更新所有预设Agent的模型配置
4. **审计记录**：记录配置变更事件

## 8. 错误处理流程
```
操作执行 → 异常捕获 → 日志记录 → 错误响应 → 审计记录
```

### 详细步骤：
1. **异常捕获**：try-catch包裹关键操作
2. **日志记录**：记录错误详情和堆栈跟踪
3. **错误响应**：返回统一格式的错误信息
4. **审计记录**：记录错误事件用于分析

## 9. 性能优化策略
1. **缓存策略**：用户信息缓存，配置缓存
2. **异步处理**：非关键操作异步执行
3. **连接池**：数据库连接池管理
4. **索引优化**：关键字段建立索引
5. **压缩传输**：API响应gzip压缩
