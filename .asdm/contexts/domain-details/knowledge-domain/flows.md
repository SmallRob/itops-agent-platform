# 知识库域业务流程

## 1. 智能检索流程
```
用户输入查询 → 解析查询参数 → 构建SQL查询 → 获取候选知识条目 → 计算相关性评分 → 排序过滤 → 返回结果
```

### 详细步骤：
1. **查询解析**：提取关键词，去除停用词
2. **候选集获取**：按分类过滤，限制50条
3. **评分计算**：5维度加权评分
   - TF-IDF相似度（0.4）
   - 精确匹配（0.3）
   - 标题匹配（0.2）
   - 使用频率（0.05）
   - 时间衰减（0.05）
4. **结果排序**：按评分降序排列
5. **使用频率更新**：更新匹配条目的使用次数
6. **高亮生成**：生成包含关键词的上下文片段

## 2. 知识注入流程（RAG）
```
用户问题 → 检索相关知识 → 格式化提示词 → 增强LLM响应
```

### 详细步骤：
1. **知识检索**：调用`search`方法获取相关知识
2. **规则引擎集成**：若无结果，调用本地规则引擎
3. **提示词生成**：
   ```
   📚 以下是从知识库中检索到的相关信息（相关度排序）：
   
   【资料 1】标题 (相关度: 85%)
   高亮片段...
   
   【资料 2】标题 (相关度: 72%)
   高亮片段...
   
   请根据以上信息和你的专业知识回答问题。
   ```
4. **使用频率更新**：增加匹配条目的使用次数

## 3. QAnything集成流程
```
用户查询 → 检查QAnything启用状态 → 构建API请求 → 重试机制 → 解析响应 → 返回上下文
```

### 详细步骤：
1. **配置检查**：验证QAnything启用状态和API密钥
2. **请求构建**：
   ```json
   {
     "user_id": "itops_agent",
     "kb_ids": ["知识库ID"],
     "question": "用户问题",
     "top_k": 5
   }
   ```
3. **重试机制**：最多2次重试，指数退避延迟
4. **响应解析**：支持字符串、数组、对象三种格式
5. **错误处理**：认证错误不重试，网络错误重试

## 4. 文档管理流程
```
上传文档 → QAnything处理 → 状态查询 → 文档删除
```

### 详细步骤：
1. **文档上传**：
   - 支持多种文件格式：PDF、Word、Excel等
   - 使用FormData上传
   - 超时设置120秒
2. **状态查询**：
   - 查询文档解析状态
   - 返回文件名和状态
3. **文档删除**：
   - 通过文件ID删除
   - 记录操作日志

## 5. 知识维护流程
```
添加知识 → 自动标签提取 → 批量导入 → 统计更新
```

### 详细步骤：
1. **知识添加**：
   - 自动生成UUID
   - 自动提取标签（关键词提取）
   - 记录创建时间
2. **批量导入**：
   - 事务处理确保一致性
   - 统计导入成功和失败数量
3. **知识更新**：
   - 更新修改时间
   - 保持使用频率不变

## 6. 配置管理流程
```
加载配置 → 缓存配置 → 配置更新 → 清除缓存
```

### 详细步骤：
1. **配置加载**：从`settings`表读取`qanything_config`
2. **配置缓存**：避免重复读取数据库
3. **配置更新**：支持运行时更新
4. **缓存清除**：更新后清除内存缓存

## 7. 错误处理流程
```
操作执行 → 异常捕获 → 日志记录 → 错误响应
```

### 详细步骤：
1. **异常捕获**：try-catch包裹关键操作
2. **日志记录**：记录错误详情和堆栈跟踪
3. **错误响应**：返回统一格式的错误信息
4. **降级策略**：QAnything不可用时回退到本地知识库

## 8. 性能优化策略
1. **查询优化**：索引优化，限制返回数量
2. **缓存策略**：配置缓存，频繁查询缓存
3. **异步处理**：非关键操作异步执行
4. **分页处理**：大数据量分页查询
